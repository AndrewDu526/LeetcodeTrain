## 2. Solution

先构造一个字母和数字的映射，tricky点在于在递归中传递index来控制数字的移动，其余是常规回溯法。注意StringBuilder的api。

## 3. Code Implementation
  
    class Solution {
        List<String> result = new ArrayList<>();
        StringBuilder path = new StringBuilder();
        String[] digit2letter = {"", "", "abc", "def", "ghi", "jkl", "mno", "pqrs", "tuv", "wxyz"};
    
        public List<String> letterCombinations(String digits) {
            backtracking(digits, 0);
            return result;
        }
    
        private void backtracking(String digits, int index){
            if(digits.length()==path.length()){
                result.add(path.toString());
                return;
            }
    
            String letters = digit2letter[digits.charAt(index)-'0'];
            
            for(int i=0;i<letters.length();i++){
                path.append(letters.charAt(i));
                backtracking(digits, index+1);
                path.deleteCharAt(path.length()-1);
            }
        }
    }

## 4. Notes

- __最初的困惑__：
  在最开始的实现中，我在每一层递归内部又写了
  for (i = 0; i < digits.length(); i++) 去遍历所有数字。
  这让我产生了几个疑问：
  我既在 for 里遍历数字，又在递归中“进入下一层”，是不是重复了？
  递归过程中为什么感觉数字又从头被遍历了一遍？  
  数字的位置推进（从第 0 位到第 1 位）到底应该放在 for 里，还是交给递归来控制？
  
  __问题的本质原因__：
  我混淆了两种完全不同的“推进方式”：
  纵向推进（递归层数）：表示正在处理第几个数字
  横向枚举（for 循环）：表示当前数字可以选择哪些字母
  把“数字的位置推进”和“字母的枚举”都交给 for，会导致递归树被错误地重复展开。

  __关键修正与正确思路__：
  digits 中“当前是第几个数字”并不是一个需要枚举的选择，而是一个按顺序推进的状态
  → 应该由递归参数 index 来控制。
  每一层递归只负责 digits[index] 这一位：
  for 负责 横向遍历 当前数字对应的所有字母
  递归调用负责 纵向推进 到 index + 1（下一个数字）
  因此，index + 1 必须写在递归调用中，而不是写在 for 的数字遍历里。
